## Aufgabenbeschreibung

1. Erstelle aus den Daten eine YAML-Datei.
2. Erstelle ein XML-File aus der JSON.
3. Validiere das XML-File mittels XSD.

### Fragen:

1. Was sind die Unterschiede der Datentypen?
    -Json - wird mit [], {}, (),  geschreieben
    -XML - wird mit <, >, geschrieben 
    -YAML - wird mit lehrzeichen geschreiben

2. Was sind Vor- und Nachteile der jeweiligen Dateiformate?
    -Json - Sehr gut lesbar          | keine Kommentare erlaubt
    -XML -  Gut validierbar          | Mehr Text was es schwer zu lesen macht
    -YAML - Lesbar, unterstützt Kommentare | einrückungen k

3. Wofür wird XML verwendet?
    -Daten bearbeitung und Speicherung

4. Unterstützt XML benutzerdefinierte Tags?
    -Ja XML ünterstützt benutzerdefinierte Tags diese müssen aber denoch bestimmten Regeln einhalten

5. Wann ist welcher Datentyp von Vorteil?
    -Bei verschiedenen größen:
    -Json für etwas kleinere Dateien und wenn Programme daten austauschen
    -XML für Große dateien und wenn die Daten länger nicht abgeändert werden müssen
    -YAML für sehr kleine dateien und wenn Menschen die Dateien lesen und schreiben müssen

6. Was ist eine XSD-Datei?
    -Sie enthält das Regelwekt wie eine xml datei aufgebaut sein darf z.B. das ein Key wort nur ein Int speichern darf

7. Was versteht man unter Validierung?
    -beim validieren prüft man dateien darauf ob sie den Regeln entspricht
